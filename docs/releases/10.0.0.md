# 10.0.0 Breaking Change Summary

Version 10.0.0 is the breaking instruction-only orchestration update shared by Claude Code and Codex.

## Changes

- Reduces `implementation-orchestrator` to one public skill file and one lifecycle reference.
- Requires read-only discovery, a proportional plan, and explicit approval before every tracked implementation; new or materially changed behavior also adds user-needs brainstorming and deeper analysis.
- Routes agents by Strong, Balanced, and Fast capabilities instead of hardcoded model identifiers.
- Keeps root coordination-only and stops before tracked edits when the active host has no writer-capable native delegation.
- Reassesses reasoning during implementation, escalating for new complexity and lowering it for deterministic segments.
- Runs targeted checks after coherent phases and one full suite after the final mutation and review, before completion and optional delivery.
- Keeps unaffected work moving when the user sends status questions or details within approved behavior, while related material changes from the same active segment share one affected-scope planning and approval cycle.
- Removes the persisted state/event contract, obsolete role/reference/task guidance, and duplicate PR/MR bridge.
- Makes the host-visible `/ant:merge-request` or `$merge-request` skill the sole PR/MR owner and keeps `/ant:delivery-workflows` or `$delivery-workflows` focused on merge conflicts.
- Builds every created or updated PR/MR description from the net target-merge-base-to-final-`HEAD` diff instead of narrating intermediate branch history.
- Updates maintainer and user documentation for the same behavior in Claude Code and Codex.

Version 10 does not read or migrate earlier custom orchestrator run files. Start a new orchestrated task after installing the update.
